2주차 학습한 내용 정리
Rest API란 웹자원 resource를 정의하여 URI(주소)와 HTTP Method(GET, POST, PUT, DELETE)를 사용하여 클라이언트와 서버가 데이터를 주고받는 방식의 설계 아키텍처 스타일을 말합니다.
HTTP 통신이란 웹 브라우저(클라이언트)와 웹 서버 간에 HTML 문서, 이미지, 영상 등 다양한 데이터를 주고받기 위한 하이퍼텍스트 전송 프로토콜(HyperText Transfer Protocol)로, 요청(Request)과 응답(Response) 방식으로 작동하며, 웹 세상의 기본 통신 규약입니다.
사용자가 웹 페이지를 요청하면, 브라우저는 서버에 HTTP 요청을 보내고, 서버는 요청받은 데이터를 HTTP 응답으로 되돌려주는 방식입니다
브라우저에 URL을입력 후 요청하여서버에서 응답하기 위해서 사용자가 웹 브라우저 주소창에 URL을 입력하면, DNS 조회를 통해 도메인 이름을 실제 서버의 IP 주소로 변환하고 TCP/IP 연결을 수립합니다. 
연결이 완료되면 브라우저는 서버에 필요한 데이터를 달라는 HTTP 요청을 보내고 서버는 이를 해석하여 JSON 등이 담긴 HTTP 응답을 다시 브라우저로 전송합니다. 마지막으로 브라우저는 전달받은 데이터를 해석하여 사용자에게 시각적인 화면으로 보여주는 과정을 수행하며 통신이 마무리됩니다.
